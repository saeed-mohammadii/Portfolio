When working with Server-Side Rendering (SSR) frameworks such as Nuxt, one concept you'll encounter frequently is **hydration**.

Hydration is the process where Vue attaches client-side JavaScript to the HTML that was already generated on the server. This allows the page to become interactive without requiring the browser to render everything from scratch.

The process usually works seamlessly, but problems can occur when the HTML generated on the server differs from what the client expects to render.

### Server renders

```html
<div>0.42</div>
```

### Client expects

```html
<div>0.87</div>
```

Since the content is different, Vue detects a mismatch during hydration and displays a hydration warning in the console.

## Why does this happen?

During hydration, Vue compares two things:

* **DOM**: The actual HTML generated by the server and sent to the browser.
* **Virtual DOM (VDOM)**: The virtual representation of the UI generated by Vue on the client.

If the server-rendered DOM and the client-generated VDOM do not match, Vue cannot hydrate the application correctly and reports a hydration mismatch.

## Common Causes

Hydration errors often occur when rendering values that can differ between the server and the client, such as:

* `Math.random()`
* `Date.now()`
* Browser-only APIs (`window`, `document`, `localStorage`)
* Data that changes between the server render and client render
* Conditional rendering based on client-specific information

For example:

```vue
<template>
  <div>{{ Math.random() }}</div>
</template>
```

The server generates one random value, while the client generates another, resulting in different HTML and a hydration mismatch.

## Key Takeaway

Hydration errors occur when Vue's Virtual DOM does not match the HTML already present in the browser. To avoid these issues, ensure that the server and client produce the same initial output during the first render.

A good rule of thumb is: if a value can be different between the server and the client, it should not be rendered during SSR without proper handling.
